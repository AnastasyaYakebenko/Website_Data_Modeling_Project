# ## Проект: Моделирование требований для интернет-магазина

**Задача:** Анализ и проектирование требований для разработки сайта интернет-магазина. В рамках проекта были разработаны функциональные требования и спроектирована структура данных.

### Разработанные артефакты:

**1. Функциональное моделирование:**
- [Use Case диаграмма и сценарии использования](https://github.com/AnastasyaYakebenko/-/blob/main/USE%20CASE_.jpg) — описание функциональных требований и взаимодействия пользователей с системой.

**2. Логическое моделирование данных:**
Проектирование структуры базы данных с последовательной детализацией и валидацией:

- [Логическая ER-диаграмма в нотации Чена](https://github.com/AnastasyaYakebenko/-/blob/main/Логическая%20ER-диаграмма%20в%20нотации%20Чена.jpg) — концептуальная модель данных, отображающая сущности и связи.
- [Логическая ER-диаграмма в нотации Crow's Foot](https://github.com/AnastasyaYakebenko/-/blob/main/Логическая%20ER%20диаграмма%20в%20нотации%20Crows%20Foot.jpg) — детализированная модель с указанием типов связей.

**3. Нормализация и оптимизация:**
- Проведена валидация ER-диаграммы в нотации Crow's Foot.
- Выполнена нормализация данных — модель приведена к **третьей нормальной форме (3НФ)**.
- [Итоговая ER-диаграмма (3НФ)](https://github.com/AnastasyaYakebenko/-/blob/main/CROWS%20FOOT.jpg) — финальная версия модели данных, готовая к передаче разработчикам.

### Ключевые навыки, продемонстрированные в проекте:
- Составление Use Case и сценариев использования
- Построение ER-диаграмм в различных нотациях (Чена, Crow's Foot)
- Нормализация данных (приведение к 3НФ)
- Валидация моделей данных
